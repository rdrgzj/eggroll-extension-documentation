Game Module

This module manages game logic, initializing level state, player states, and gameplay flow. It integrates level management, movement logic, and utility functions.

Module Dependencies

Built-in Modules: os, time

Third-party Libraries: termcolor

Project-specific Modules: level_manager, movement, utility

Class(es) and Function(s)

class game.Game(level_file: str)

Bases: object

Handles the entire gameplay flow, including level setup, player input, movement and score mechanics, and game completion. It also manages transitions like restarting the game or returning to the main menu.

display_game_state() → None

Displays the current state of the game.

Calls the degridder function from the utility module to render the grid. Displays the current level, number of moves played, remaining moves, and score. Also shows options for special inputs like restarting or returning to the main menu.

end_game_menu() → None

Displays the end game menu.

Shows options for the next level, restarting the game, or returning to the main menu, and waits for user input.

end_game_menu_input() → None

Handles the input for the end game menu.

Asks the player for their choice, directing them to the next level, restarting the game, or returning to the main menu based on their input.

end_game_state() → None

Displays the final game state.

Shows the current grid, moves played, final score, and the leaderboard before transitioning to the end game menu.

get_player_input() → tuple[str, list[str]]

Asks the user for input, validates it against valid directions (‘L’, ‘R’, ‘F’, ‘B’), and returns both the raw input and the list of valid moves. This method utilizes the MovementManager.dirs for valid directions.

handle_special_input(dir_input: str) → bool

Checks if the input contains ‘X’ (restart) or ‘Y’ (main menu), and returns True to indicate a special action. Calls restart_game and return_to_main_menu as needed.

high_scorer() → None

Handles high score logic.

If the player has achieved a high score, asks for their name and updates the leaderboard with the new score.

is_high_scorer() → bool

Checks if the player beats a high score in the leaderboard.

Loads the leaderboard scores from Leaderboard.load_scores and compares the player’s score to the current top scores.

play() → None

Main gameplay loop.

Displays the current game state and processes player input until all moves are used or there are no eggs left in the grid. Ends the game and checks for high score eligibility.

play_next_level() → None

Handles the next level logic.

Determines the next level based on the current level’s file name, sets up the new level, and restarts the gameplay loop. If no next level is available, informs the player and returns to the main menu.

process_valid_moves(valid_moves: list[str]) → None

Handles the actual movement of eggs.

For each valid move in the input, it moves the eggs according to the direction, updates the score using Player.update_score, and decreases the number of moves left. Calls MovementManager.move_until_blocked to perform the actual move. Displays the updated game state after each move.

restart_game() → None

Resets the level, player, and movement manager, then restarts the gameplay loop.

return_to_main_menu() → None

Handles the transition to the main menu, importing it only when needed to avoid circular dependencies.

class game.Level(level_file: str)

Bases: object

The Level class initializes the level based on a provided file, counts the eggs and nests, calculates the score, and checks for remaining eggs in the grid.

calc_score() → int

Calculates the current score based on nests collected and eggs removed.

It compares the initial count of eggs and nests with the current grid state to compute the score.

count_eggs_and_nests() → int

Iterates through the grid to count the occurrences of eggs (’🥚’) and nests (’🪺’).

has_eggs() → bool

Iterates through the grid to check if any row contains an egg (’🥚’).

class game.Player(total_moves: int, eggs: int)

Bases: object

The Player class tracks the player’s remaining moves, previous moves, and score. It also updates and finalizes the score during the game.

decrement_moves() → None

Decrease the moves_left attribute by 1, used after each valid move.

finalize_score() → None

Finalizes the player’s score at the end of the game.

Adds remaining moves as bonus points and prints the bonus if moves are left.

update_score(level: Level) → None

Updates the player’s score based on the current grid state.

Uses the Level class to calculate the current score and assigns it to the player’s score attribute.
